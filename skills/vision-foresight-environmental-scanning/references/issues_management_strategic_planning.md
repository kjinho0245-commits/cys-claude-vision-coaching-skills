# 환경 스캐닝 ↔ Issues Management ↔ Strategic Planning 통합 framework

> Gordon & Glenn (2009) Appendix D + Renfro (1993) *Issues Management in Strategic Planning* 통합

## 3-tier 통합 도식

환경 스캐닝은 *고립된 활동*이 아니다. Issues Management 사이클의 첫 단계이자, Strategic Planning의 외부 관점 루프다.

```
┌──────────────────────────────────────────────────────────────────┐
│                     EXTERNAL PERSPECTIVE                         │
│                  (ENVIRONMENTAL SCANNING)                        │
│                                                                  │
│   Evaluating·Ranking ──► Forecasting ──► Scanning ──► Monitoring │
│                                                       │          │
│                                                       ▼          │
└────────────────────────────────────────────────────── │ ─────────┘
                                                        │
                                                        ▼
┌──────────────────────────────────────────────────────────────────┐
│                  STRATEGIC PLANNING PROCESS                      │
│                                                                  │
│        Forecasting ◀─────────────────────────► Forecasting       │
│                                                                  │
└──────────────────────────────────────────────────── │ ───────────┘
                                                      │
                                                      ▼
┌──────────────────────────────────────────────────────────────────┐
│                     INTERNAL PERSPECTIVE                         │
│                   (LONG RANGE PLANNING)                          │
│                                                                  │
│        Goal Setting ──► Forecasting ──► Implementing             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 핵심: 두 루프가 *Forecasting 단계에서 교차*

External (Environmental Scanning) loop는 *외부 변화*를 예측. Internal (Long Range Planning) loop는 *내부 목표·실행*을 예측. 두 루프가 Forecasting에서 만나 *strategic decisions*를 형성.

### 핵심 통찰

> "Environmental scanning and issues management are approaches to help large organizations and nations manage their future. To work, **they must be embedded in a network reaching throughout the nation**." — Joseph F. Coates, *Issues Management* (1986)

— *조직 전체에 걸친 네트워크*로 embedded되어야 작동. 단독 부서 활동으로는 실패.

## NLTPS (National Long Term Perspective Studies) 적용

### NLTPS 5 phases (Renfro 4-stage cycle의 국가 단위 확장)

NLTPS는 25-30년 horizon의 국가 전략 기획 프로세스. 환경 스캐닝이 첫 phase의 핵심.

1. **Issue Identification** — 환경 스캐닝으로 emerging issues 식별
2. **Preparing the Base of Study** — 과거·현재 분석, actors·dynamics 매핑
3. **Constructing Scenarios** — 대안 미래 시나리오 구축
4. **Designing Alternative Strategies** — 시나리오별 전략 설계
5. **Strategic Agenda and Action Plan** — 정부 단위 액션 플랜

### NLTPS 환경 스캐닝 위원회 구성 (PDF 권고)

- **15명 environmental scanning committee**
- **4명**: National Core Team (정부 기획부서)
- **11명**: 외부 전문가·visionary (대학·NGO·multidisciplinary working groups)
- **회의 빈도**: 초기 월 1회, 안정 후 격월 1회
- **의사결정**: 합의제 (secret balloting로 그룹 dominance 방지)

## QUEST와의 관계

QUEST (Quick Environmental Scanning Technique)는 환경 스캐닝의 *축약형 워크숍 버전*. NLTPS의 Issue Identification phase 안에서 QUEST를 단기 워크숍으로 운영 가능.

→ 본 통합 framework 안에서 QUEST는 *Issue Identification 단계의 facilitation 도구*로 위치.

→ QUEST 상세는 **vision-foresight-environmental-scanning-quest-workshop** 스킬로 위임.

## OPIN Program 사례 (Ohio Policy Issues Network)

PDF가 인용한 *환경 스캐닝의 issues analysis 확장* 모범 사례.

### OPIN 핵심 8 questions (Issue Analysis 가이드)

각 emerging issue에 대해 다음 8개 질문 답변:

1. What is **new** about the issue?
2. What specific **facts** are known that substantiate this emerging issue?
3. What further **information**, if any, is needed to support or confirm this issue?
4. How is this issue **relevant** to local leaders, state officials, and governors?
5. How can local leaders, governors, and other policy makers **influence** this issue?
6. Who are the other **actors** this issue will affect?
7. What other **levels of government** will this issue impact?
8. What **policy options** can we propose and what comments do we have about them?

→ 본 8 questions은 **vision-foresight-environmental-scanning-issues-management** 스킬의 Stage 2 (Researching) 단계에서 활용.

## Strategic Planning과의 시간 sync

### 연간 사이클 권고 (PDF·Renfro 권고)

```
1월: 신년 환경 스캐닝 priority 설정
2-3월: 전 분기 결과 종합·issues 식별
4월: 1차 Issues Management Committee
5-6월: Issue Research + Impact Wheel
7월: Probability-Impact Chart 평가
8월: Top 3-5 Consensus Issues 결정
9월: Strategy Selection
10월: Strategic Plan 통합 (Long Range Planning)
11-12월: 차년도 priority + 시스템 calibration
```

### 박사님 컨텍스트 적용 (한국 미래학 연구소)

- 1월: 박사님 신년 단행본 주제 + 박사 강의 캘린더 사전 priority
- 분기별: 박사님 자문위원회 (외부 expert 5~10명)
- 연간: 박사님 신년 단행본·트렌드 보고서 출간 → feedback → 차년도 스캐닝 priority


- 1월: 신년 사역 주제 + 비전 캐스팅 priority
- 분기별: 당회 + 평신도 panel 정기 회의
- 연간: 당회 사역 회고 → 차년도 사역 계획 → 환경 스캐닝 calibration
