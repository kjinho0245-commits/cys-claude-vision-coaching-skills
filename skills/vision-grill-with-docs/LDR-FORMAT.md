# LDR (Life Decision Record) 포맷 사양

본 문서는 `vision-grill-with-docs` 인터뷰 도중 내려진 *되돌리기 어려운 인생 결정*을 기록하는 포맷을 규정한다. 원본 `grill-with-docs/ADR-FORMAT.md`의 ADR(Architecture Decision Record)을 미래비전코칭 맥락의 LDR로 매핑했다.

---

## 1. 위치·번호

LDR은 사용자 작업 폴더의 `docs/ldr/` 아래 순차 번호로 저장한다.

```
<사용자 작업 폴더>/
└── docs/ldr/
    ├── 0001-진로-공학에서-신학으로-전환.md
    ├── 0002-결혼-2027-가을-결단.md
    └── 0003-사역-단기선교-3년.md
```

- 번호는 4자리 영문 숫자 (`0001`, `0002`, ...).
- 슬러그는 *간결한 한글 케밥*: `진로-공학에서-신학으로-전환` 형식. 띄어쓰기는 하이픈, 조사 생략.
- `docs/ldr/` 디렉토리는 첫 LDR이 필요한 시점에 lazy 생성. 빈 폴더 미리 만들지 않는다.
- 번호 부여는 `grill_lib.py next_ldr_number --base <작업폴더>`가 자동 처리 (기존 최대값 + 1).

---

## 2. 최소 템플릿 (한 단락이면 충분)

```markdown
# {결정 제목 — 짧고 명확하게}

**날짜**: 2026-05-18
**영역**: 진로 | 재정 | 관계 | 사역 | 건강 | 비전 영역 전환

{1~3문장: 어떤 맥락에서 무엇을 결정했고 왜 결정했는가.}
```

이게 전부다. **LDR은 한 단락이어도 된다.** 가치는 *결정이 내려졌다는 사실*과 *왜*를 남기는 데 있지, 섹션을 채우는 데 있지 않다.

### 최소 예시

```markdown
# 진로 — 공학에서 신학으로 전환

**날짜**: 2026-05-18
**영역**: 진로

2026년 봄 학기 부전공 결정 시점에 공학 단일 전공 → 신학 부전공 추가로 결정. 박사님 표준 사전 *소명* 정의(시대적 부르심에 응답한 일)에 본인 응답이 신학 영역에서 더 분명했다는 판단. 졸업 시점 신학 진학 가능성도 열어 둔 결정.
```

---

## 3. 선택 섹션 (진짜 가치 있을 때만)

대부분 필요 없다. 다음 셋은 추가 가치가 있을 때만 붙인다.

### 3-A. Status (의사결정 상태)

결정이 시간 따라 바뀔 가능성이 있을 때:

```markdown
**Status**: proposed | accepted | deprecated | superseded by LDR-NNNN
```

- `proposed`: 인터뷰 도중 잠정 결단 — 사용자가 24~48시간 더 숙고하기로 함
- `accepted`: 사용자가 명시적으로 결정 확정
- `deprecated`: 시간이 지나 결정이 무효화됨 (사유 1줄)
- `superseded by LDR-NNNN`: 후속 LDR이 이 결정을 대체

### 3-B. Considered Options (고려한 대안)

거절한 대안을 6개월 뒤에 "왜 안 했지?" 되묻지 않도록 기록할 가치가 있을 때:

```markdown
## 고려한 대안
- 공학 단일 전공 유지: 5년 후 안정성은 높으나 박사님 *소명* 정의에 본인 응답이 약함.
- 신학으로 전과: 가족 합의 미달 + 부모 재정 지원 불확실. 보류.
- *선택: 부전공 추가* — 두 진로 모두 열어두는 비용 합리적.
```

### 3-C. Consequences (장기 영향)

코드(=인생 일정)에 안 보이는 downstream 효과가 있을 때만:

```markdown
## Consequences
- 졸업 학기 1학기 연장 가능성.
- 부모 재정 부담 1.2배 증가 추정 — `vision-financial-coach` 산출물에 반영 필요.
- 5년 후 신학 진학 진로 가능 — `vision-futures-timeline-map`에 신규 분기 추가.
```

## 4. LDR 발행 3조건 — `grill_lib.py check_ldr_criteria`로 자동 판정

결정을 LDR로 만들기 *전에* 반드시 3조건 모두 충족 여부 검사. 하나라도 빠지면 LDR 만들지 않는다.

### 조건 1. Hard to reverse (되돌리기 어렵다)

번복 비용이 크다. *비용*은 시간·돈·관계·기회·정서 모두 포함.

판정 가이드:
- O 해당: 결혼 결단, 집 매수/매도, 직장 이직, 이주, 사역 헌신, 출산, 대학 진학, 전공 변경
- X 미해당: 자가용 색상 선택, 이번 주 식단, 헬스장 등록, 한 학기 과목 선택

### 조건 2. Surprising without context (맥락 없이 보면 놀라움)

5년·10년 뒤 본인이 "왜 그때 그렇게 결정했지?" 의문을 가질 결정이거나, 이 결정을 모르는 제3자(미래 자녀·미래 코칭 후속자·미래 본인)가 보면 "왜 의외의 선택을 했지?"라고 묻게 될 결정.

판정 가이드:
- O 해당: 안정 진로 포기하고 창업, 부모 반대 무릅쓰고 신학 진학, 박사 학위 도중 사역으로 전환
- X 미해당: 평범한 회사원 진로 유지, 일반 결혼 일정, 일반 부동산 거래

### 조건 3. Result of a real trade-off (진짜 대안이 있었다)

진짜 대안이 있었고 의도해서 한 선택. 자동 결정·관성 결정·"그렇게 할 수밖에 없었다"는 LDR 대상이 아니다.

판정 가이드:
- O 해당: A·B·C 셋 다 가능했고 비교해서 A를 골랐다.
- X 미해당: 다른 선택지가 없었다, 자동으로 그렇게 됐다, 결정 안 하면 자동 디폴트로 갔을 결정.

### 자동 판정 호출

```bash
python3 grill_lib.py check_ldr_criteria \
    --decision "공학에서 신학으로 부전공 추가" \
    --reversibility hard \
    --surprising true \
    --tradeoff true
```

출력: `{"qualifies": true, ...}` 또는 `{"qualifies": false, "missing": ["surprising"]}`.

3조건 미충족이면 LDR 만들지 말고 사용자에게 통지:

> "이 결정은 LDR 3조건 중 [surprising]을 충족하지 않아 LDR 기록 대상이 아닙니다. VISION-CONTEXT.md 본인 맥락 정의에는 반영했습니다. LDR 없이 진행해도 후속 인터뷰에서 문제 없습니다."

## 5. 자격 있는 결정 유형 (미래비전코칭 맥락)

박사님 vision 시리즈에서 LDR 대상이 되는 전형적 결정들:

### 진로
- 전공 변경·전과·부전공 추가
- 이직·창업·휴직·휴학
- 직장 → 신학교, 일반 → 사역, 사역 → 일반 전환

### 재정
- 집 매수/매도, 부동산 큰 거래
- 자녀 유학 결정
- 창업 자본 투입
- 부채 큰 약정 (주담대·사업 대출)
- 큰 증여·상속 처리

### 관계
- 결혼 결단·이혼·재혼
- 자녀 출산 결정
- 부모 부양 형태 결정
- 거주 분리·합가

### 사역
- 선교사 헌신 (단기 vs 장기)
- 교회 개척
- 전임 사역 vs 이중직
- 안수·신학 진학
- 직장 사역자 헌신

### 건강
- 큰 수술·치료 방향 (대안 비교)
- 운동·식습관·금주 같은 *6개월 이상 지속*하는 약속 (이 경우는 조건 1 검토 필요)

### 비전 영역 전환
- `vision-three-realm-balance` 결과에서 무게 중심 이동
- 박사님 비전 정의(*가치 있는 시대적 소명*) 구성 요소(*가치*·*시대*·*소명*) 중 어느 한 자리의 본인 답이 바뀐 경우

### 시대적 소명 응답
- 박사님 본인 미래학자 본업·집필 결단·강의 헌신 등 메타 결정

## 6. LDR 작성 시 톤

- **한 단락이면 충분.** 채우려고 늘리지 마라.
- **"왜"가 핵심.** "무엇을 했는가"는 한 줄이면 충분. "왜 그렇게 했는가"는 *맥락 + 박사님 사전 매핑 + 거절한 대안*까지 들어가야 미래 본인이 읽을 때 의미가 있다.
- **박사님 표준 사전 용어 사용.** 본인 맥락 용어는 VISION-CONTEXT.md § 2에 정의된 것만.
- **이모지 금지.**
- **할루시네이션 차단** — 박사님 책 인용은 SOURCES.md에 명시된 문장만.

---

## 7. 발행 절차 요약

1. 사용자 결정 감지 → `grill_lib.py check_ldr_criteria` 호출 → 3조건 통과 확인.
2. `grill_lib.py next_ldr_number --base <작업폴더>` → 다음 번호 부여.
3. 사용자에게 LDR 제안 + 초안(한 단락 최소 템플릿) 보여주기.
4. 사용자 확인·수정 → 파일 작성 (`docs/ldr/NNNN-슬러그.md`).
5. 인터뷰 계속.

---

## 8. LDR가 *아닌* 것 (오해 차단)

- LDR은 **다이어리가 아니다**. 일상 결정 기록이 아니라 *되돌리기 어려운* 결정 기록.
- LDR은 **목표 설정이 아니다**. 비전 선언문은 VISION-CONTEXT.md § 본인 맥락에. 목표·미션 진술은 `vision-mission-frame` 또는 `vision-statement-writer` 영역.
- LDR은 **숙고일지가 아니다**. 결정이 *내려진* 시점에 발행. 숙고 중인 사항은 인터뷰 진행으로 grill.
- LDR은 **고백록·신앙고백이 아니다**. 신앙 묵상은 sermon 시리즈 영역. LDR은 *결정의 사실·왜·맥락*만.
