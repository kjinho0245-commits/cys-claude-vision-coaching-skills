# Workshop Facilitation Tools — 10 도구 활용 가이드

> Gordon-Glenn (2009) PDF Appendix C가 QUEST 또는 active environmental scanning에서 활용 권고한 10가지 도구

## PDF 원전

> "In doing QUEST or any other active environmental scanning various tools can be used to solicit emerging issues which are of importance to the organization. **Questionnaires, Delphi, SWOT analysis, stakeholder analysis, futures wheels, probability charts, assumption testing, structural analysis, cross impacts analysis, and scenario planning** are among the various procedures which can be used."

## 10 Tools — Workshop 적용 가이드

### Tool 1 — Questionnaires (사전 설문)

#### 적용 시점
- Phase 1 Preparation 단계
- Workshop 사전 1~2주

#### 권고 양식
- 5~10 questions, mix of:
  - Open-ended (3~5)
  - Likert scale (3~5)
  - Ranking (1~2)

#### 박사님 컨텍스트 — 사전 설문 sample

```markdown
# QUEST Pre-Workshop Survey

## Q1 (Open): 우리 institution이 직면한 가장 중요한 미래 이슈 3개?
1. ...
2. ...
3. ...

## Q2 (Open): 5년 후 우리 institution이 어떻게 변할 것 같은가?
[2~3 sentences]

## Q3 (Likert 1-5): 다음 trends의 중요도
- AGI 출현: 1·2·3·4·5
- 한국 인구절벽: 1·2·3·4·5
- 한국 기독교 인구 변화: 1·2·3·4·5
- (etc.)

## Q4 (Ranking): 다음 시나리오 중 가장 plausible 순서
- A: 한국 일본화
- B: 한국 부분 회복
- C: 통일 한반도
- D: 분단 지속
[Ranking 1~4]

## Q5 (Open): 우리 institution의 가장 큰 strategic blind spot은?
```

### Tool 2 — Delphi

#### 적용 시점
- Phase 1·2·3 (다중 라운드 가능)
- 시간 제약 시 *Conversational Delphi* 변형

#### Delphi protocol (formal)
- Round 1: Open question
- Round 2: Cross-reference (다른 panelist 응답에 대한 평가)
- Round 3: Convergence + outlier justify

#### Conversational Delphi (workshop 친화)
- 1 round: 익명 평가 → median + IQR display → discussion → re-vote
- Workshop 내 60~90분에 가능

#### 박사님 컨텍스트 적용
- 박사님 자문위원 panel에 Delphi 정기 운영
- 카카오톡·Notion 활용 가능

### Tool 3 — SWOT Analysis

#### 적용 시점
- Phase 2 Workshop Segment 1 (Mission 토의 직후)
- 또는 Phase 4 Strategic Options 평가 시

#### 표준 매트릭스

```
              Helpful              Harmful
            ┌────────────┬────────────┐
   Internal │  Strengths │  Weaknesses │
            ├────────────┼────────────┤
   External │ Opportunities│   Threats │
            └────────────┴────────────┘
```

#### TOWS 확장 (Strategic Options 도출)
- SO: Strengths × Opportunities → 공격
- ST: Strengths × Threats → 방어
- WO: Weaknesses × Opportunities → 보완
- WT: Weaknesses × Threats → 회피·축소

#### 박사님 컨텍스트 SWOT 사례

##### 미래학 연구소
- **Strengths**: 박사님 brand·통합적 (영성+미래학)·30년 경력
- **Weaknesses**: 1인 의존·글로벌 노출 제한·디지털 인프라
- **Opportunities**: AGI 시대 의미 추구·한국 미래학 수요 증가·해외 진출
- **Threats**: 미래학자 노화·신선한 thinkers 등장·AI 자동화

### Tool 4 — Stakeholder Analysis

#### 적용 시점
- Phase 1 Preparation
- Phase 2 Workshop Segment 1
- Phase 3 Report Part 1

#### 표준 매트릭스 (Power × Interest)

```
                    Interest
                Low         High
              ┌──────────┬───────────┐
   High Power │ Keep     │  Manage   │
              │Satisfied │ Closely   │
              ├──────────┼───────────┤
   Low Power  │ Monitor  │ Keep      │
              │          │ Informed  │
              └──────────┴───────────┘
```

#### Stakeholder Mapping 단계
1. *모든* potential stakeholders list (15~30명)
2. Power score (1~10)
3. Interest score (1~10)
4. 4 분면 분류
5. 분면별 engagement strategy

#### 박사님 컨텍스트 사례

##### 미래학 연구소 stakeholders
- **Manage Closely (High×High)**: 박사님 가족·핵심 자문위원·정기 청중
- **Keep Satisfied (High×Low)**: 정부 정책 관계자·신학 동료·자문 client
- **Keep Informed (Low×High)**: 박사님 ardent 독자·학계 동료
- **Monitor (Low×Low)**: 일반 미래학 관심층

### Tool 5 — Futures Wheels

#### 적용 시점
- Phase 2 Workshop Segment 2·3·4
- Phase 3 시나리오 implications 작성

#### 통합
- **vision-foresight-futures-wheel** 스킬 직접 호출
- STEEPS 6차원 + 세옹지마 비선형
- 1차·2차·3차·4차

#### Workshop 내 Futures Wheel 진행
1. (10분) 중앙 trend 1개 선정
2. (15분) Small groups (3~4명) — 각 group이 STEEPS 1차원 1차 영향 도출
3. (15분) 그룹 share·합치기
4. (15분) 2차 영향 가지치기

### Tool 7 — Assumption Testing

#### 적용 시점
- Phase 2 Workshop Segment 1·2 (mission·trends 토의 직후)

#### Protocol
1. 그룹의 *implicit assumptions* 식별
2. 각 assumption *명시적*으로 진술
3. *Counter-evidence* 도전
4. *Sensitivity* — 이 assumption이 깨지면 무엇이 변하나?

#### 박사님 컨텍스트 — 흔한 assumption (도전 필요)
- "한국 미래학 시장이 계속 성장한다"
- "박사님 brand가 자동 유지된다"
- "한국 기독교 인구는 점진 감소만 한다"
- "AGI는 5년 내 본격 침투"
- "한반도 통일은 향후 30년 어렵다"

### Tool 8 — Structural Analysis

#### 적용 시점
- Phase 2 Workshop Segment 3 (cross-impact 후)
- Phase 3 Report — 시스템 dynamics 정리

#### Protocol — MICMAC 변형
1. Variables list (10~20)
2. Direct influence matrix
3. Indirect influence (square 매트릭스)
4. *Dependent vs influencing* variables 식별
5. *Driving forces* (high influence·low dependence)
6. *Result variables* (low influence·high dependence)

#### Workshop 활용
- Cross-impact analysis 이후 *시스템 구조* 시각화
- Whiteboard에 driving forces·result variables 매트릭스

              Driver 2 ↑

    ┌───────────┼───────────┐

- Structural Analysis (Tool 8) — 시스템 dynamics 깊은 분석 시

## Facilitation 팁

### 시간 관리
- 각 segment *strict* timing
- 휴식 사이사이 (90분 이상 일하지 않음)
- 모든 segment에 *명확한 output* 정의

### 그룹 dynamics
- *Quiet voices* 명시적 격려
- *Dominant voices* polite redirect
- Small group → big group 흐름 활용

### 시각화
- Whiteboard·flipchart 적극
- Sticky notes 색깔 분류
- 사진·기록 (consent 확인)

### 박사님 컨텍스트 특수 고려
- 박사님 본인이 *facilitator + participant* 이중 역할 시: 외부 facilitator (자문위원 1명) 부탁
- 또는 박사님이 *participant 역할*에서는 1차 의견 표시 *마지막*에
- Anchoring·dominance 방지
