## 자동화 권고 — 사용자 cmux 환경

### Generation pipeline
1. **Records aggregation** — vision-foresight-environmental-scanning-weak-signal-template DB에서 월간 records 추출
2. **Domain classification** — 자동 (LLM 기반)
3. **Item selection** — Top 5 strategic items (LLM + 사용자 review)
4. **Implications generation** — 사용자 3 컨텍스트 (LLM + 사용자 review)
5. **Sources extraction** — records의 sources 자동 정리
6. **Item 7 cross-reference** — issue ID matching·timeline 자동 update
7. **Item 8 selection** — 신간 외부 보고서 monitoring (자동 alert + 사용자 confirm)
8. **Appendix expansion** — 사용자 review 후 deep dive items 결정
9. **Final review** — 사용자 final read·revision

→ Steps 1·2·5·6·7 자동화. Steps 3·4·8·9 사용자 review.

### LLM·Claude 활용
- Records → Item summary: Claude·Notion AI
- Implications draft: Claude (사용자 컨텍스트 prompt)
- Sources 정리: Python 스크립트
- Cross-reference: 사용자 cmux markdown DB grep

### Time investment 추정
- 자동 generation: 1~2 시간 (인프라 setup 후)
- 사용자 review·revision: 2~4 시간
- Final: 4~6 시간 per monthly report
